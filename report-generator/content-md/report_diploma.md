# ВВЕДЕНИЕ

Актуальность работы обусловлена интересом к симуляторам профессиональной деятельности. Симуляторы позволяют безопасно осваивать навыки управления техникой. Это особенно важно для профессий водного транспорта. Разработка симулятора судоводителя на Unity объединяет современные подходы к физическому моделированию и игровому проектированию. Реализованы реалистичная физика и экономическая составляющая.

Объект работы -- процесс интерактивного управления трёхмерной моделью судна. Предмет -- методы расчёта гидростатических и гидродинамических сил, действующих на корпус судна, и проектирование экономической подсистемы.

Проблема -- отсутствие доступных локальных симуляторов судоводителя с реалистичной физикой корпуса на волне и экономической механикой. Цель -- разработка игры-симулятора судоводителя с реалистичной физикой на волновой поверхности и упрощённым экономическим контуром.

Задачи работы:

1. анализ предметной области;
2. разработка технического задания;
3. проектирование архитектуры;
4. создание приложения на Unity;
5. тестирование и оценка экономической эффективности.

Теоретическая значимость -- систематизация методов расчёта гидростатических и гидродинамических сил применительно к игровому движку Unity. Практическая значимость -- создание локального симулятора как наглядного пособия при изучении основ управления водным транспортом.

Методы исследования: анализ предметной области, математическое моделирование физических процессов и компонентное проектирование. Для реализации применялось объектно-ориентированное программирование на C#. Тестирование выполнено экспериментально на разработанном приложении.

Структура работы включает введение, разделы, заключение, список источников и приложения.

## 1.2. Назначение и цели создания системы

### 1.2.1. Назначение системы

Система «BoatSim» -- локальное однопользовательское приложение-симулятор. Оно предназначено для демонстрации принципов управления быстроходным судном с водомётным движителем на волновой поверхности. Приложение может использоваться как наглядное пособие при изучении основ управления водным транспортом и физики движения тел в жидкостях.

### 1.2.2. Цели создания системы

Целями создания системы являются:

1. разработка реалистичной физической модели движения судна на волновой поверхности с расчётом гидростатических и гидродинамических сил;
2. реализация игровой механики управления водомётным движителем;
3. создание упрощённого экономического контура с операциями покупки и продажи товаров;
4. обеспечение наглядного отображения эксплуатационных параметров: скорости, времени и баланса.

### 1.2.3. Задачи создания системы

Для достижения поставленных целей разработаны следующие компоненты:

1. физическая модель корпуса с расчётом плавучести, сопротивления и ударных нагрузок;
2. механика водомётного движителя с подачей тяги, поворотом сопла и автоматическим возвратом в нейтраль;
3. экономический контур магазина с покупкой и продажей товаров;
4. интерфейс отображения скорости, времени и баланса с переключением режимов: обзор, причал, судно;
5. волновая поверхность с визуальными эффектами: брызги и след;
6. игровое время с ускорением и сменой суток;
7. обзорная камера с вращением и масштабированием.

## 1.3. Требования к системе

### 1.3.1. Требования к функциональным характеристикам

#### 1.3.1.1. Требования к составу выполняемых функций

Система выполняет следующие функции:

1. обработка команд клавиатуры для движения и поворота;
2. расчёт физических сил корпуса: плавучести, сопротивления и ударных нагрузок;
3. моделирование волновой поверхности с динамическим изменением высоты волн;
4. расчёт и отображение скорости;
5. моделирование игрового времени;
6. ведение экономического контура;
7. переключение режимов интерфейса;
8. управление обзорной камерой;
9. визуализация эффектов: брызги, след и вращение;
10. корректное завершение по команде пользователя.

#### 1.3.1.2. Требования к организации входных данных

Входными данными системы являются сигналы клавиатуры:

1. клавиша W увеличивает мощность двигателя;
2. клавиши A и D поворачивают сопло влево и вправо;
3. клавиши E и F переключают режимы интерфейса;
4. клавиша Escape завершает приложение.

Дополнительные входные данные включают клики по кнопкам интерфейса магазина для покупки и продажи товаров.

#### 1.3.1.3. Требования к организации выходных данных

Система выводит:

1. текущую скорость судна в км/ч или узлах;
2. игровое время и номер дня;
3. денежный баланс;
4. окно магазина с ценами и количеством товаров;
5. трёхмерную визуализацию водной поверхности, судна и эффектов.

#### 1.3.1.4. Требования к взаимодействию с пользователем

Взаимодействие с пользователем осуществляется через клавиатуру и графический интерфейс:

3. завершение работы -- клавиша Escape;

## 3.2. Создание компонентов компьютерной модели

В качестве основных компонентов приложения разработаны:

1. `EngineController.cs` -- модуль управления двигателем с подачей тяги, поворотом сопла и учётом расхода топлива;
2. `BoatPhysics.cs` -- модуль физики корпуса с расчётом сил плавучести, вязкостного сопротивления, сопротивления давления и ударных нагрузок;
3. `BuoyancyManager.cs` -- менеджер плавучести для погружения треугольных сегментов корпуса;
4. `WaveGenerator.cs` -- модуль водной поверхности для вычисления высоты волны в произвольной точке;
5. `WaterManager.cs` -- компонент иллюзии бескрайней водной поверхности;
6. `ShopManager.cs` -- модуль магазина с покупкой и продажей товаров;
7. `TimeScript.cs` -- компонент игрового времени с ускорением 60:1;
8. `Speedometer.cs` -- компонент отображения скорости;
9. `UIManager.cs` -- компонент переключения режимов интерфейса.

## 3.3. Написание основного программного кода

### 3.3.1. Модуль управления двигателем

Листинг 1 демонстрирует фрагмент кода модуля управления двигателем.

![Листинг 1. Фрагмент кода модуля управления двигателем](../input/engine.cs.png)

В каждом шаге физического расчёта запас топлива уменьшается пропорционально текущей мощности. Когда топливо заканчивается, мощность принудительно обнуляется и судно останавливается.

### 3.3.2. Модуль физики корпуса

Трёхмерная модель корпуса разбивается на треугольные сегменты, и для каждого сегмента определяется, находится ли он ниже поверхности воды. Если сегмент погружён, к нему прикладываются силы плавучести, вязкостного сопротивления, сопротивления давления и ударной нагрузки. Сила плавучести рассчитывается как произведение плотности воды, ускорения свободного падения и объёма погружённой части. Силы сопротивления пропорциональны квадрату скорости, площади сегмента и плотности среды. Ударная нагрузка возникает при быстром входе корпуса в воду и добавляется к общей сумме сил.

### 3.3.3. Модуль водной поверхности

Водная поверхность реализована через два взаимодополняющих компонента. Первый вычисляет высоту волны в произвольной точке пространства на основе синусоидальных функций с шумовыми искажениями. Полученные параметры передаются в шейдер для визуализации. Второй компонент создаёт иллюзию бескрайней водной поверхности: генерируется сетка квадратов, которая перемещается вслед за камерой таким образом, что край воды никогда не виден.

### 3.3.4. Модуль магазина

Магазин хранит данные о товарах в двумерном массиве. Для каждого товара определены идентификатор, цена и количество. При покупке проверяется достаточность средств: если баланс позволяет, сумма списывается, а количество товара увеличивается. При продаже выполняется обратная операция.

### 3.3.5. Модуль времени

## 4.2. Расчёт себестоимости одного машинного часа работы ЭВМ

# РАЗДЕЛ V. МЕРОПРИЯТИЯ ПО ТЕХНИКЕ БЕЗОПАСНОСТИ И ОХРАНЕ ТРУДА

## 5.1. Параметры микроклимата

Рабочее место пользователя компьютера должно соответствовать определённым санитарно-гигиеническим требованиям, обеспечивающим комфортную и безопасную работу.

Температура воздуха:

В холодный период года -- от 22 до 24 °C.

В тёплый период года -- от 23 до 25 °C.

Относительная влажность воздуха:

Должна поддерживаться в диапазоне от 40% до 60%. Это предотвращает пересушивание воздуха, которое может вызывать дискомфорт, а также снижает риск забоев слизистой оболочки.

Скорость движения воздуха:

В холодный период -- не более 0,1 м/с.

В тёплый период -- не более 0,2 м/с. Увлажнение и оптимальная температура воздуха снижают вероятность перегрева организма и излишнего сухого воздуха, что положительно сказывается на работоспособности.

Нормативный документ:

СанПиН 1.2.3685-21 «Гигиенические нормативы факторов рабочей среды и трудового процесса».

## 5.2. Освещённость рабочего места

Освещённость рабочего места имеет большое значение для комфортной работы с компьютером и предотвращения перенапряжения глаз, головных болей и других негативных последствий.

Естественное освещение должно быть сбалансированным и не создавать прямых солнечных лучей, попадающих на экран монитора. Это может вызывать бликов, что затрудняет восприятие информации.

Искусственное освещение:

Освещённость на рабочем месте должна составлять от 300 до 500 лк. Это количество света достаточно для комфортного восприятия информации с экрана.

Коэффициент пульсации освещения -- не более 10%. Это необходимо для предотвращения усталости глаз и ухудшения зрения.

Расположение оборудования:

Экран монитора должен быть размещён так, чтобы свет падал сбоку (перпендикулярно к экрану), что исключает появление бликов и улучшает восприятие изображения.

Нормативные документы:

СанПиН 2.2.2/2.4.1340-03, СП 52.13330.2016 «Естественное и искусственное освещение».

## 5.3. Обеспечение электробезопасности

Электробезопасность является ключевым аспектом в защите работников от поражения электрическим током при работе с электрическим оборудованием.

Компьютерная техника должна подключаться через розетки с заземляющим контактом, что обеспечивает защиту от возможных электрических замыканий.

Устройства защитного отключения (УЗО) обязаны быть установлены при групповых подключениях, что помогает избежать поражения током в случае неисправности оборудования.

Все провода и разъёмы должны быть:

с заводской изоляцией,

без механических повреждений,

уложены таким образом, чтобы исключить риск спотыкания и возникновения травм.

Запрещается:

использовать удлинители без заземления и самодельные удлинители, так как это увеличивает риск электрических аварий.

Нормативные документы:

ПУЭ («Правила устройства электроустановок»), ГОСТ 12.1.030-81 «Электробезопасность. Защита от статического электричества».

## 5.4. Обеспечение пожарной безопасности

Предотвращение пожаров и обеспечение безопасности сотрудников в случае возникновения чрезвычайных ситуаций требует соблюдения ряда правил и мер.

Оснащение помещений должно быть огнетушителями (порошковыми или углекислотными), а также системами противопожарной сигнализации, которые помогут быстро обнаружить и ликвидировать очаг возгорания.

Наличие плана эвакуации и указателей направлений выхода критически важно в случае возникновения пожара или других аварийных ситуаций.

Регулярная проверка электропроводки и оборудования на отсутствие перегрева и искрения позволяет избежать коротких замыканий и других опасных ситуаций.

Запрещается:

загромождать эвакуационные пути и выходы, что может препятствовать быстрой эвакуации в случае аварии.

Использовать неисправные электроприборы, так как это может стать источником пожара.

Нормативные документы:

Федеральный закон №123-ФЗ «Технический регламент о требованиях пожарной безопасности», Постановление Правительства РФ №1479 «Правила противопожарного режима».

## 5.5. Требования охраны труда перед началом работы

Перед началом работы пользователь должен проверить оборудование и создать оптимальные условия для выполнения задач, чтобы минимизировать риск травм или ухудшения самочувствия.

Проверка целостности оборудования:

Убедиться, что все элементы компьютера (монитор, клавиатура, мышь) работают исправно.

Проверить все провода и разъёмы на наличие повреждений.

Корректировка рабочего места:

Настроить высоту кресла и монитора для правильного положения тела.

Монитор должен быть на уровне глаз, а расстояние до экрана -- 50--70 см.

Спина должна быть прямой, а стопы располагаться на полу или подставке для ног.

Проверить рабочее место на наличие посторонних предметов, чтобы избежать спотыкания или случайных травм.

Нормативный документ: СанПиН 2.2.2/2.4.1340-03.

## 5.6. Требования охраны труда во время работы

Во время работы важно следить за состоянием здоровья, соблюдать правила гигиены труда и устранять возможные причины усталости.

Регламентированные перерывы:

После 1 часа работы -- перерыв 5--10 минут для восстановления.

При 8-часовом рабочем дне общая продолжительность перерывов должна составлять минимум 50--60 минут.

Упражнения для глаз, шеи и рук помогут избежать зрительного утомления, болей в спине и руках.

Правильная посадка:

Следить за посадкой, чтобы не допустить искривления позвоночника или перенапряжения мышц.

Немедленно сообщать о неисправностях оборудования ответственному лицу для минимизации возможных аварийных ситуаций.

Нормативный документ: СанПиН 2.2.2/2.4.1340-03.

## 5.7. Требования охраны труда в аварийных ситуациях

В случае аварийной ситуации (например, возгорания или поражения электрическим током), необходимо строго следовать установленным правилам безопасности:

Обесточить оборудование незамедлительно для предотвращения дальнейших аварий.

Использовать огнетушитель в случае возгорания.

Эвакуироваться согласно заранее подготовленному плану эвакуации.

Оказать первую помощь пострадавшему и вызвать медицинскую помощь по телефону 112.

# ПРИЛОЖЕНИЕ Б. ИНСТРУКЦИЯ АДМИНИСТРАТОРА

## Б.1. Системные требования

Для установки и запуска приложения «BoatSim» необходимо:

- процессор -- Intel Pentium IV / 1.8 ГГц или выше;
- объём оперативной памяти -- 512 МБ RAM;
- место на жёстком диске -- 100 МБ HDD;
- видеокарта с поддержкой DirectX 10;
- операционная система -- Windows 7 и выше.

## Б.2. Установка приложения

Установка приложения выполняется в три шага:

1. скопировать папку `BoatSim` на целевой компьютер.
2. запустить файл `BoatSim.exe`.
3. при первом запуске возможно предупреждение системы безопасности Windows. Необходимо разрешить запуск.

## Б.3. Настройка параметров

Параметры волновой поверхности настраиваются через редактор Unity. Доступны следующие параметры:

- `WaveScale` -- масштаб волны;
- `WaveSpeed` -- скорость распространения волн;
- `WaveDistance` -- расстояние между гребнями;
- `NoiseAmount` -- интенсивность шумовых искажений.

# ПРИЛОЖЕНИЕ В. ИНСТРУКЦИЯ ПОЛЬЗОВАТЕЛЯ

Руководство пользователя для приложения «BoatSim».

## В.1. Введение

Приложение «BoatSim» -- это игра-симулятор профессии судоводителя с реалистичной физикой движения судна и упрощённым экономическим контуром. Пользователь управляет быстроходным судном с водомётным движителем на волновой поверхности, отслеживает эксплуатационные параметры и выполняет торговые операции.

## В.2. Навигация по приложению

Приложение состоит из следующих элементов управления:

- клавиши W, A, D -- управление движением и поворотом судна;
- клавиши E, F -- переключение режимов интерфейса;
- клавиша Escape -- завершение приложения;
- интерфейс магазина -- покупка и продажа товаров.

## В.3. Как управлять судном

Управление движением:

- нажмите W для подачи тяги вперёд;
- нажмите A для поворота сопла влево;
- нажмите D для поворота сопла вправо;
- отпустите A или D -- сопло автоматически вернётся в нейтральное положение.

## В.4. Режимы интерфейса

Интерфейс состоит из трёх режимов, переключаемых клавишами E и F:

- режим мирового обзора отображает скорость судна, игровое время, номер дня и денежный баланс поверх трёхмерной сцены. Все элементы расположены в верхней части экрана.
- режим причала открывает окно магазина с перечнем товаров, их ценой и количеством. Нажмите «Купить» или «Продать» для выполнения соответствующей операции.
- в каждый момент времени активен только один режим, что предотвращает наложение окон.

## В.5. Частые вопросы

**Как узнать текущую скорость судна?**
Скорость отображается в левом верхнем углу экрана в режиме мирового обзора. Значение приводится в км/ч или узлах.

**Что делать, если судно перестало двигаться?**
Проверьте запас топлива. При исчерпании топлива двигатель автоматически отключается. При необходимости перезапустите приложение.

**Как завершить работу приложения?**
Нажмите клавишу Escape. Приложение завершит работу корректно.

Режим причала открывает окно магазина с перечнем товаров, их ценой и количеством. Пользователь может нажать кнопку «Купить» или «Продать» для выполнения соответствующей операции.

# ПРИЛОЖЕНИЕ Д. СТРУКТУРА СИСТЕМЫ

## Г.1. Состав файлов проекта

```
Assets/
  Scripts/
    EngineController.cs     // Модуль управления двигателем
    BoatPhysics.cs          // Физика корпуса
    BuoyancyManager.cs      // Менеджер плавучести
    WaveGenerator.cs        // Генератор волн
    WaterManager.cs         // Менеджер водной поверхности
    ShopManager.cs          // Модуль магазина
    TimeScript.cs           // Игровое время
    Speedometer.cs          // Отображение скорости
    UIManager.cs            // Управление интерфейсом
  Scenes/
    BoatSim.unity           // Основная сцена
  Prefabs/
    Boat.prefab             // Префаб судна
    Water.prefab            // Префаб водной поверхности
```

## Г.2. Алгоритм работы приложения

Алгоритм работы приложения состоит из следующих шагов:

1. при запуске загружается сцена `BoatSim.unity`;
2. инициализируются все компоненты: двигатель, физика, водная поверхность, интерфейс;
3. запускается игровой цикл:
  1. обработку ввода с клавиатуры;
  2. расчёт физики корпуса;
  3. обновление водной поверхности;
  4. обновление интерфейса;
  5. визуализация сцены.
4. при нажатии клавиши Escape приложение завершает работу.
